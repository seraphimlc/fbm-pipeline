# Excel And Decision Schema

## Expected Input

The workbook can contain many product rows, but each run must target exactly one row or one folder ID. The scripts search header names flexibly.

Single-product red lines:

- Do not create an index containing multiple products.
- Do not scan the workbook sibling directory to discover all product folders.
- Resolve only the current row's folder ID, then inspect only `<image_root>/<folder_id>`.
- If the target row/ID cannot be matched, stop and write a concise row-level remark instead of looking through other product folders.

Preferred columns:

- Product title: `标题`, `商品标题`, `title`, `product title`
- Five bullets: `bullet1`-`bullet5`, `卖点1`-`卖点5`, or columns containing `bullet`/`卖点`
- Warehouse URL or folder ID: `大件云仓`, `云仓链接`, `warehouse url`, `url`, `图片链接`, `ID`

The final ID in the URL is used to match an ID-named folder under the image root.

## Output Files In Each ID Folder

Full image analysis should not be stored in Excel because long multimodal text makes the workbook slow and hard to read. For each matched product row, write a subfolder under the ID-named product folder:

```text
<workbook sibling image root>/<folder_id>/image analysis/
├── image_selling_points.json   # complete cached image review and selection data
├── image_selling_points.md     # human-readable per-image selling-point analysis
├── gallery_selection.md        # selected 01-09 gallery slots and reasons
├── aplus_planner_input.json    # structured downstream input for Amazon A+ Image Planner
└── aplus_planner_input.md      # concise downstream input for Amazon A+ Image Planner
```

Use `image_selling_points.json` as the authoritative machine-readable cache. Use `aplus_planner_input.md` or `.json` as the handoff into Amazon A+ Image Planner.

## Lightweight Output Columns On Source Sheet

Excel should only keep pointers and short summaries:

- `图片分析文件夹`: absolute path to `<folder_id>/image analysis/`.
- `A+图片规划输入文件`: absolute path to `aplus_planner_input.md`.
- `图片卖点摘要`: short human-readable Chinese summary.
- `主图选择摘要`: short selected-gallery summary, not full JSON.
- `当前工作节点`: set to `6-图片分析` after successful image analysis export.
- `处理备注`: concise error/status note when the run fails or needs manual attention.

Do not write full per-image multimodal JSON or long gallery selection JSON into Excel cells by default. Only create full Excel analysis tabs when the user explicitly asks for an Excel-only deliverable.

## Decision JSON Shape

```json
{
  "products": [
    {
      "product_key": "folder-or-row-id",
      "row_number": 2,
      "folder_id": "12345",
      "listing_intent": {
        "product_type": "",
        "variant": "",
        "buyer_problem": "",
        "visible_claims": [],
        "caution_claims": []
      },
      "image_reviews": [],
      "gallery_selection": [
        {
          "slot": "01",
          "image_id": "#01",
          "filename": "source.jpg",
          "path": "/absolute/source.jpg",
          "selected_selling_points": ["product_identity"],
          "score": 90,
          "selection_reason": ""
        }
      ],
      "missing_recommended_images": []
    }
  ]
}
```

`analysis_decisions_template.json` generated by the scanner follows this shape and can be filled manually after multimodal review.
