# FBM铺货系统 — 架构设计

> 版本：0.1.0 | 更新日期：2026-05-13

---

## 1. 系统概述

FBM铺货是一套亚马逊 FBM（自发货）商品上架自动化系统。输入一个大健云仓（GigaB2B）商品链接，系统自动完成从**商品采集→利润计算→关键词调研→类目分析→Listing生成→主图分析→A+页面规划→A+脚本编写→A+出图**的9步流水线，最终产出可直接上架的亚马逊商品资料。

### 核心价值

- **一键铺货**：从大健云仓链接到完整 Amazon Listing，9步全自动
- **AI驱动**：GPT-5.5 生成 Listing 文案 + A+内容，Qwen VLM 分析主图，GPT Image 生成 A+ 图片
- **优雅降级**：Step 2/3/4 缺少前置条件时自动跳过（非阻断），不阻塞后续步骤
- **本地部署**：全部运行在本地 Mac，无需云服务器

## 2. 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| **后端** | Python 3.12+ / FastAPI | 异步 REST API |
| **数据库** | SQLite (aiosqlite) | 轻量级本地存储，4张表 |
| **ORM** | SQLAlchemy 2.0 (async) | 异步 ORM + migration |
| **前端** | React 19 + TypeScript + Ant Design | SPA 管理界面 |
| **构建** | Vite | 前端构建工具 |
| **LLM** | OpenAI-compatible API (sub2api) | GPT-5.5，用于 Listing/A+ |
| **VLM** | DashScope (通义千问) | qwen3.6-plus，用于主图分析 |
| **Image** | GPT Image (t8star) | gpt-image-2，用于 A+ 出图 |
| **浏览器** | Chrome + AppleScript | JS注入控制，用于大健云仓采集 |
| **数据源** | GigaB2B + SellerSprite | 商品源 + 关键词反查 |

## 4. 数据模型

### 4.1 ER 关系

```
products (1) ──→ (0..1) product_data      ← 商品基础数据 + Listing数据
products (1) ──→ (0..1) product_images    ← 图片分析结果
products (1) ──→ (0..1) product_aplus     ← A+内容数据
```

每张子表通过 `product_id` 外键一对一关联 `products`。

### 4.2 表结构概览

#### products（商品主表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int PK | 自增主键 |
| gigab2b_url | text | 大健云仓商品链接 |
| gigab2b_product_id | string(50) | GigaB2B 商品编号（如 W3327） |
| competitor_asin | string(20) | 竞品 ASIN（用于关键词反查） |
| brand | string(100) | 品牌（默认 Vindhvisk） |
| status | string(30) | 当前管道状态 |
| current_step | int | 当前步骤编号（0-9） |
| error_message | text | 错误信息 |
| created_at / updated_at | datetime | 时间戳 |

#### product_data（商品数据 — 9步数据汇总）

这张表承载了 Step 1~5 的全部输出结果，按模块分区：

- **模块1（采集）**：item_code, title, color, material, dimension_*, weight, packages, features, variants, stock, seller, origin, image_count, material_dir, value_total, estimated_total
- **模块2（定价）**：suggested_price, cost_total, profit, profit_rate
- **模块3（关键词）**：keywords_top (JSON), keyword_excel_path
- **模块4（类目）**：categories (JSON), leaf_category
- **模块5（Listing）**：listing_title, listing_bullets (JSON), listing_search_terms, listing_check (JSON), listing_primary_keyword, listing_removed_keywords (JSON)

#### product_images（图片分析）

- **VLM分析**：contact_sheet_path, image_analysis (JSON), image_selling_points (JSON), category_style
- **主图选择**：main_image_path, main_image_source, gallery_images (JSON), gallery_order (JSON), main_image_summary
- **元信息**：analyzed_at, vlm_model

#### product_aplus（A+内容）

- **A+规划**：aplus_plan (JSON), aplus_plan_summary
- **A+脚本**：aplus_scripts (JSON), aplus_scripts_summary
- **A+出图**：aplus_images (JSON), aplus_image_count, aplus_status
- **时间戳**：planned_at, scripted_at, generated_at

## 5. Pipeline 引擎

### 5.1 执行流程

```
Step1(采集) → Step2(定价) → [Step3(关键词) ∥ Step4(类目)] → Step5(Listing) → Step6(主图) → Step7(A+规划) → Step8(A+脚本) → Step9(A+出图)
              └─ 非阻断 ──┘  └──── 非阻断，可并行 ────┘
```

### 5.2 状态机

每个商品在 Pipeline 中经历以下状态：

```
created
  → step1_collecting → step1_done
  → step2_pricing    → step2_done
  → step3_keywords ─┐
  │                  ├→ step3_4_done
  → step4_category ─┘
  → step5_listing   → step5_done
  → step6_curating  → step6_done
  → step7_aplus_plan  → step7_done
  → step8_aplus_script → step8_done
  → step9_aplus_image  → step9_done
  → completed

  # 异常状态
  → failed    (任何步骤出错)
  → paused    (用户手动暂停)
```

### 5.3 优雅降级策略

| 步骤 | 跳过条件 | 影响 |
|------|---------|------|
| Step 2（定价） | 大健云仓未登录/无价格数据 | 无建议售价，其余正常 |
| Step 3（关键词） | 未提供 competitor_asin 或无 SellerSprite Token | 无关键词数据，Step5 用基础信息生成 Listing |
| Step 4（类目） | 未提供 competitor_asin | 无类目建议，需手动填 |

关键设计：**Step 2/3/4 失败不阻断 Pipeline**，异常被捕获后标记完成，继续执行后续步骤。

### 5.4 并行执行

Step 3（关键词）和 Step 4（类目）通过 `asyncio.gather()` 并行执行，二者无数据依赖。

### 5.5 任务管理

- **后台执行**：Pipeline 通过 `asyncio.create_task()` 异步运行，不阻塞 API 响应
- **单例锁**：每个 product_id 同一时间只允许一个 Pipeline 实例运行（`_running_tasks` 字典管理）
- **取消/暂停**：通过 `task.cancel()` 发送 `CancelledError`，Pipeline 在 `except` 中将状态设为 `paused`
- **重试**：从失败的步骤重新执行，不清除已完成步骤的数据

## 6. 外部依赖

### 6.1 依赖关系矩阵

| 步骤 | 外部依赖 | 必须? | 失败策略 |
|------|---------|-------|---------|
| Step 1（采集） | Chrome 浏览器 + GigaB2B 网页 | ✅ | 阻断 |
| Step 2（定价） | GigaB2B 价格数据 | ❌ | 跳过 |
| Step 3（关键词） | SellerSprite API Token + ASIN | ❌ | 跳过 |
| Step 4（类目） | ASIN | ❌ | 跳过 |
| Step 5（Listing） | LLM API (sub2api) | ✅ | 阻断 |
| Step 6（主图） | VLM API (DashScope) | ✅ | 阻断 |
| Step 7（A+规划） | LLM API (sub2api) | ✅ | 阻断 |
| Step 8（A+脚本） | LLM API (sub2api) | ✅ | 阻断 |
| Step 9（A+出图） | GPT Image API (t8star) | ✅ | 阻断 |

### 6.2 Chrome 控制

系统通过 AppleScript 控制本地 Chrome 浏览器（非 CDP），提供三个核心操作：

- `chrome_navigate(url)` — 导航到指定 URL
- `chrome_execute_js(code)` — 注入并执行 JavaScript
- `chrome_click_element(selector)` — 通过 CSS 选择器点击元素

所有 Chrome 操作通过 `asyncio.Lock()` 串行化，同一时间只有一个操作执行。

**前提条件**：Chrome 菜单 → 视图 → 开发者 → ✅ 允许 Apple 事件中的 JavaScript

### 6.3 AI API 配置

| 用途 | 代理地址 | 模型 | 协议 |
|------|---------|------|------|
| LLM（Listing/A+） | sub2api.127space.com/v1 | gpt-5.5 | OpenAI-compatible |
| VLM（主图分析） | dashscope.aliyuncs.com | qwen3.6-plus | OpenAI-compatible |
| GPT Image（A+出图） | ai.t8star.cn/v1 | gpt-image-2 | OpenAI-compatible |

所有 API 通过 `.env` 配置 Key，通过 `/api/config` 接口可查看配置状态（Key 是否已填写）。

## 9. 配置管理

所有配置通过 `.env` 文件 + `Settings` 类管理（`pydantic-settings`），支持环境变量覆盖。

关键配置项：

| 配置 | 默认值 | 说明 |
|------|--------|------|
| BACKEND_PORT | 8190 | 后端端口 |
| FRONTEND_PORT | 3190 | 前端端口 |
| DEFAULT_BRAND | Vindhvisk | 默认品牌名 |
| LLM_API_KEY | — | GPT-5.5 API Key（必须） |
| VLM_API_KEY | — | Qwen VLM API Key（必须） |
| GPT_IMAGE_API_KEY | — | GPT Image API Key（必须） |
| SELLERSPRITE_TOKEN | — | 卖家精灵 Token（可选） |
| STEP3_4_PARALLEL | true | Step3/4 是否并行 |
| APLUS_CONCURRENCY | 5 | A+ 图片生成并发数 |
