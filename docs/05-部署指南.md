## 1. 环境要求

### 硬件

| 项目 | 最低要求 | 推荐 |
|------|---------|------|
| 内存 | 4 GB | 8 GB+ |
| 磁盘 | 1 GB 可用 | 5 GB+（图片存储） |
| 网络 | 稳定互联网 | 低延迟国际线路 |

### 软件

| 软件 | 版本 | 用途 |
|------|------|------|
| **Python** | 3.11+ | 后端运行时 |
| **Node.js** | 18+ | 前端构建 |
| **Google Chrome** | 最新版 | Step 1/3/4 浏览器自动化 |
| **macOS** | 13+ (Ventura) | AppleScript 自动化依赖 |

> ⚠️ **macOS 限定**：系统使用 AppleScript 控制 Chrome，目前仅支持 macOS。Linux/Windows 需要改造 Chrome 控制模块。

### 外部服务

| 服务 | 用途 | 必需 |
|------|------|------|
| 大健云仓（GigaB2B）账号 | Step 1 商品采集 | ✅ |
| 卖家精灵（SellerSprite）账号 | Step 3 关键词调研 | 可选 |
| 亚马逊卖家账号 | 最终上架 | 可选（系统不上架） |
| LLM API（sub2api） | Step 5/7/8 | ✅ |
| VLM API（DashScope） | Step 6 | ✅ |
| GPT Image API（t8star） | Step 9 | ✅ |

## 2. 安装步骤

### 2.1 获取代码

```bash
cd ~/code
git clone <repo-url> fbm-pipeline
cd fbm-pipeline
```

### 2.2 后端安装

```bash
cd ~/code/fbm-pipeline/backend

# 创建虚拟环境
python3 -m venv .venv
source .venv/bin/activate

# 安装依赖
pip install -r requirements.txt
```

**后端依赖清单**（`requirements.txt`）：

| 包 | 版本 | 用途 |
|----|------|------|
| fastapi | 0.115.12 | Web 框架 |
| uvicorn[standard] | 0.34.2 | ASGI 服务器 |
| sqlalchemy | 2.0.41 | ORM |
| aiosqlite | 0.21.0 | 异步 SQLite 驱动 |
| pydantic | 2.11.3 | 数据验证 |
| pydantic-settings | 2.9.1 | 配置管理 |
| httpx | 0.28.1 | 异步 HTTP 客户端 |
| python-multipart | 0.0.20 | 文件上传 |
| openpyxl | 3.1.5 | Excel 解析（Step 3） |
| openai | (间接) | LLM/VLM/GPT Image 客户端 |
| Pillow | (间接) | 图片处理（Step 6 Contact Sheet） |

### 2.3 前端安装

```bash
cd ~/code/fbm-pipeline/frontend

# 安装依赖
npm install
```

**前端依赖**（`package.json` 核心项）：

### 2.4 macOS 权限配置

FBM 铺货系统使用 AppleScript 控制本地 Chrome，需要以下权限：

#### 2.4.1 自动化权限
```
系统设置 → 隐私与安全 → 自动化 → ✅ 勾选终端 / ClawX / VS Code
```

#### 2.4.2 辅助功能权限
```
系统设置 → 隐私与安全 → 辅助功能 → ✅ 勾选终端 / ClawX
```

#### 2.4.3 Chrome JavaScript 权限
```
Chrome 菜单栏 → 视图 → 开发者 → ✅ 允许 Apple 事件中的 JavaScript
```

> ⚠️ 每次更新 Chrome 后可能需要重新勾选此选项。

#### 2.4.4 屏幕录制权限（可选）
```
系统设置 → 隐私与安全 → 屏幕录制 → ✅ 终端 / ClawX
```
截图/视觉识别需要此权限，当前系统未使用但预留给未来扩展。

## 3. 配置

### 3.1 创建环境变量文件

```bash
cd ~/code/fbm-pipeline/backend
cp .env.example .env
```

### 3.2 编辑 .env 文件

```bash
# === AI API 配置（必填） ===

# LLM - 用于 Listing 生成、A+ 规划、A+ 脚本
LLM_API_BASE=https://sub2api.127space.com/v1
LLM_API_KEY=sk-your-llm-key-here
LLM_MODEL=gpt-5.5

# VLM - 用于图片分析
VLM_API_BASE=https://dashscope.aliyuncs.com/compatible-mode/v1
VLM_API_KEY=sk-your-vlm-key-here
VLM_MODEL=qwen3.6-plus

# GPT Image - 用于 A+ 出图
GPT_IMAGE_API_BASE=https://ai.t8star.cn/v1
GPT_IMAGE_API_KEY=sk-your-image-key-here
GPT_IMAGE_MODEL=gpt-image-2

# === 卖家精灵（可选） ===
SELLERSPRITE_TOKEN=              # JWT Token，约 24h 有效

# === 项目配置 ===
DEFAULT_BRAND=YourBrandName      # 默认品牌名
BACKEND_PORT=8190                # 后端端口
FRONTEND_PORT=3190               # 前端端口

# === Chrome ===
CHROME_LOCK_TIMEOUT=300          # Chrome 操作超时（秒）

# === Pipeline ===
STEP3_4_PARALLEL=True            # Step 3/4 并行执行
APLUS_CONCURRENCY=5              # A+ 出图并发数
```

### 3.3 配置优先级

```
环境变量 > .env 文件 > config.py 默认值
```

### 3.4 Chrome 登录态准备

Step 1 需要从大健云仓采集商品，确保：
1. Chrome 已登录 GigaB2B（`www.gigab2b.com`）
2. 登录态未过期
3. Chrome 进程正在运行

Step 3 需要 SellerSprite Cookie：
1. Chrome 已登录 SellerSprite（`www.sellersprite.com`）
2. 或在 .env 中配置 `SELLERSPRITE_TOKEN`

### 3.5 商品存储目录

默认路径：`~/Documents/F/亚马逊工作目录/亚马逊商品/大健云仓/`

```
大健云仓/
└── {Brand}/
    └── {W-ID}/
        ├── image1.jpg        # Step 1 下载的原图
        ├── image2.jpg
        ├── contact_sheet.jpg # Step 6 Contact Sheet
        └── new aplus image/  # Step 9 A+ 出图
            ├── aplus_01.png
            ├── aplus_02.png
            └── ...
```

如需修改，设置环境变量 `MATERIAL_BASE_DIR`。

## 4. 启动与停止

### 4.1 一键启动

```bash
cd ~/code/fbm-pipeline
./scripts/start.sh
```

输出：
```
📦 FBM Pipeline 启动中...
🔧 启动后端 (port 8190)...
🎨 启动前端 (port 3190)...

✅ FBM Pipeline 已启动!
   前端: http://localhost:3190
   后端: http://localhost:8190
   API文档: http://localhost:8190/docs
```

### 4.2 分别启动

**后端**：
```bash
cd ~/code/fbm-pipeline/backend
source .venv/bin/activate
uvicorn app.main:app --host 0.0.0.0 --port 8190
```

**前端**（开发模式）：
```bash
cd ~/code/fbm-pipeline/frontend
npm run dev
```

**前端**（生产构建）：
```bash
cd ~/code/fbm-pipeline/frontend
npm run build
npm run preview
```

### 4.3 停止服务

一键脚本模式下按 `Ctrl+C` 停止所有服务。

分别启动模式下，各自终端 `Ctrl+C` 停止。

### 4.4 后台运行（可选）

```bash
# 使用 nohup 后台运行
cd ~/code/fbm-pipeline/backend
source .venv/bin/activate
nohup uvicorn app.main:app --host 0.0.0.0 --port 8190 > /tmp/fbm-backend.log 2>&1 &

# 查看日志
tail -f /tmp/fbm-backend.log

# 停止
kill $(lsof -ti:8190)
```

## 6. 常见问题

### Q1: `Chrome 导航失败`

**原因**：Chrome 未运行或 AppleScript 权限未授予。

**解决**：
1. 确保 Chrome 正在运行
2. 系统设置 → 隐私与安全 → 自动化 → 勾选终端
3. Chrome 菜单 → 视图 → 开发者 → 允许 Apple 事件中的 JavaScript

### Q2: `SSL: CERTIFICATE_VERIFY_FAILED`

**原因**：sub2api 等第三方 API 的 SSL 证书问题。

**解决**：系统已内置 SSL 绕过（`httpx.AsyncClient(verify=False)`），通过 `settings.get_llm_client()` 获取的客户端自动处理。

### Q3: `SellerSprite Token 过期`

**原因**：JWT Token 约 24 小时过期。

**解决**：
1. 重新登录 SellerSprite
2. 通过 Chrome Cookie 重新获取（系统自动）
3. 或手动更新 .env 中的 `SELLERSPRITE_TOKEN`

### Q4: `No image data returned` (GPT Image)

**原因**：API 返回格式不符或配额耗尽。

**解决**：
1. 检查 `GPT_IMAGE_API_KEY` 是否有效
2. 检查 API 余额
3. 查看 `ai.t8star.cn` 服务状态

### Q5: Step 2/3/4 被 Skip

**这是正常行为**。当缺少必要输入时，这些步骤会 graceful skip：
- Step 2：缺 `cost_price`
- Step 3：缺 `competitor_asin` 或 `SELLERSPRITE_TOKEN`
- Step 4：缺 `competitor_asin`

### Q6: 前端无法连接后端

**检查**：
1. 后端是否启动：`curl http://localhost:8190/api/config/status`
2. 端口是否被占用：`lsof -i :8190`
3. 前端 API 配置：`frontend/src/api/index.ts` 中的 `baseURL`

### Q7: 权限问题（macOS）

如果看到 "Not allowed to send Apple events" 弹窗：
1. 打开 系统设置 → 隐私与安全 → 自动化
2. 勾选对应应用（终端/ClawX/VS Code）
3. 重启终端后重试
