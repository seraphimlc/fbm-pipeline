# Domain Index README

`docs/domain-index/` 存放领域地图。领域地图只回答“这个问题应该去哪里看”，不回答“完整怎么设计、为什么这么设计、历史上怎么讨论”。

领域地图是路牌，不是说明书、PRD、changelog、handoff 或问题记录。

## 使用方式

1. 先读 `docs/project-index.md` 判断问题属于哪个领域。
2. 再读对应 domain index。
3. 用 domain index 指定范围执行 scoped `rg` 或读取关键片段，核实当前事实。
4. 如果发现索引过期，任务结束时更新索引；如果只是局部 bug fix 且入口/语义/验证方式没变，不更新。

## 当前领域

- `task-runtime.md`：任务中心、新任务框架、旧离线任务边界。
- `product-flow.md`：商品列表、详情、图片、竞品、商品状态和 TikTok/Amazon 详情分流。
- `export-flow.md`：Amazon 导出、导入模板、类目映射、导出任务。
- `data-sources.md`：数据源、GIGA 拉品、库存/价格同步。
- `frontend-pages.md`：前端路由、页面入口、API client 和页面验证入口。
- `runtime-security.md`：启动边界、本地访问保护、TLS、文件/图片代理。

## 固定栏目

领域地图只能使用以下栏目，避免失控：

```markdown
# Domain Index: <Name>

## 范围
## 当前口径
## 关键入口
## 关键流程
## 相关文档
## 验证入口
## 常见定位
## 维护规则
```

## 内容边界

只写三类内容：

- 去哪看：页面、API、service、worker/action、表、配置、验证入口。
- 别走错：范围、不包含什么、当前稳定口径。
- 怎么继续查：PRD/spec/review/QA/handoff 链接和简短定位指引。

不要写：

- 普通 bug 修复记录、任务列表、历史聊天、协作过程。
- 长设计解释、完整字段表、长 SQL、代码片段、长测试输出。
- 临时脚本、一次性调试命令、不稳定猜测。
- 和索引定位无关的背景、复盘、争议或“某某说过”。

## 长度约束

- `范围` 最多 5 条。
- `当前口径` 最多 8 条。
- `关键流程` 最多 5 条，每条一行。
- `相关文档` 最多 10 个链接。
- `验证入口` 最多 8 条。
- `常见定位` 最多 8 条。
- 单个领域地图建议不超过 150 行。

## 更新触发

必须更新：

- 新增、删除、迁移页面/API/核心 service/action/table。
- 任务类型、状态语义、操作矩阵、导出链路、外部集成或验证入口变化。
- 原索引指向过期文件、漏掉关键入口或误导定位。

不需要更新：

- 局部 bug fix，且不改变入口、语义、状态、表或验证方式。
- 函数内部重构、样式微调、文案微调、测试补充。
