# 06 - 试用稳定版操作说明

这版先按“能稳定批量跑、人工知道看哪里、商品状态不乱”来用。不要追求全自动，先跑一段真实商品，记录卡点后再迭代。

## 每天打开先看哪里

1. 看“商品生成任务”顶部快捷状态：
   - `运行中`：正在跑，先不用处理
   - `待人工处理`：需要补登录、补类目、补采集问题
   - `失败`：需要看错误后重试
   - `可确认`：Step10 已完成，等人工确认入库

2. 看“商品列表”顶部快捷状态：
   - `可同步ASIN`：已入库但还没有真实 ASIN
   - `ASIN未查到` / `ASIN多匹配`：系统不会自动写 ASIN，人工处理
   - `可上传A+`：已有真实 ASIN，A+ 还没传
   - `A+失败`：进入上传记录看失败原因
   - `上架高风险`：先复核 Step10 的上架检查

## 跑批前检查

- 商品创建/导入时尽量填完整：原始链接、竞品 ASIN、UPC。
- 如果要批量启动，只选择 `待启动` 的任务。
- 多任务会按系统配置并发执行；浏览器相关步骤会排队，避免多个任务同时抢浏览器。

## 人工处理规则

- Step3 提示登录：手动登录卖家精灵，然后点继续。
- Step4 缺类目：手动填 Amazon 类目和叶子类目，然后点继续。
- Step10 完成后：先看“导入表格检查”和风险提醒，再确认入库。
- 不可售、下架、库存 0 的商品不继续收集，通常不用处理。

## 商品入库后的规则

- 只有人工确认入库后的商品才进入“商品列表”。
- ASIN 同步只在领星 Listing 明确唯一匹配时写入真实 ASIN。
- 查不到 ASIN：标记 `未查到`，不自动处理。
- 多匹配：标记 `多匹配`，不自动处理。
- A+ 上传需要真实 ASIN；没有 ASIN 的商品会跳过并提示。

## 失败后怎么处理

- 生成任务失败：先看商品详情错误信息，再用“重试”。
- ASIN 同步失败：看 ASIN 同步记录，常见是领星未登录或页面加载异常。
- A+ 上传失败：看 A+ 上传记录，常见是缺 ASIN、缺 5 张 A+ 图、图片尺寸不合格、领星登录失效。
- 高风险商品：不要急着上传，先检查主图、库存、利润、关键字段和 Listing 风险。

## 哪些状态可以先不管

- `运行中`：等待完成。
- `已提交` 的 A+：后续看领星/亚马逊审核结果。
- `不可售`：采集阶段已经停止。
- `未查到` / `多匹配`：系统不会乱写，需要人工决定是否补 ASIN。

## 试用期记录

试用时重点记三类问题：

- 哪一步人工介入最多
- 哪种失败最常见
- 商品列表还缺什么筛选或状态

这三类问题比继续加大自动化更重要，下一版就按真实使用痛点收敛。
