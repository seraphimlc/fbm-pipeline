# FBM铺货系统 — 配置说明

> 版本：0.1.0 | 更新日期：2026-05-13

---

## 1. 配置体系

系统使用 **pydantic-settings** 管理配置，支持3层优先级（高→低）：

1. **环境变量**（最高优先级）
2. **`.env` 文件**（`backend/.env`）
3. **代码默认值**（`config.py` 中的 `Settings` 类）

配置加载后通过 `settings = Settings()` 全局单例使用。

---

## 2. 配置文件

### 2.1 `.env` 文件

位置：`~/code/fbm-pipeline/backend/.env`

```bash
# ─── LLM API (sub2api — Listing/A+规划/A+脚本) ───
LLM_API_BASE=https://sub2api.127space.com/v1
LLM_API_KEY=sk-your-llm-key-here
LLM_MODEL=gpt-5.5

# ─── VLM API (通义千问 DashScope — 主图分析) ───
VLM_API_BASE=https://dashscope.aliyuncs.com/compatible-mode/v1
VLM_API_KEY=sk-your-vlm-key-here
VLM_MODEL=qwen3.6-plus

# ─── GPT Image API (t8star — A+出图) ───
GPT_IMAGE_API_BASE=https://ai.t8star.cn/v1
GPT_IMAGE_API_KEY=sk-your-image-key-here
GPT_IMAGE_MODEL=gpt-image-2

# ─── 卖家精灵 Token（Cookie中的 Sprite-X-Token）───
SELLERSPRITE_TOKEN=
```

> ⚠️ `.env` 包含敏感 Key，已加入 `.gitignore`，不入版本库。

### 2.2 环境变量覆盖

任何配置项都可通过环境变量覆盖，例如：

```bash
export LLM_API_KEY=sk-new-key
export BACKEND_PORT=9000
uvicorn app.main:app --port $BACKEND_PORT
```

## 3. 配置项详解

### 3.1 项目基础

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `PROJECT_NAME` | str | "FBM Pipeline" | 项目名称 |
| `VERSION` | str | "0.1.0" | 版本号 |
| `DEBUG` | bool | True | 调试模式 |

### 3.2 数据库

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `DATA_DIR` | Path | ~/code/fbm-pipeline/data | 数据目录 |
| `DATABASE_URL` | str | 自动生成 | SQLite 连接串（无需手动配置） |

`DATABASE_URL` 自动生成规则：
```
sqlite+aiosqlite:///{DATA_DIR}/fbm.db
```
即默认为 `~/code/fbm-pipeline/data/fbm.db`。

### 3.3 服务端口

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `BACKEND_PORT` | int | 8190 | 后端 API 端口 |
| `FRONTEND_PORT` | int | 3190 | 前端开发服务器端口 |

### 3.4 文件存储

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `PRODUCT_BASE_DIR` | Path | ~/Documents/F/亚马逊工作目录/亚马逊商品/大健云仓 | 商品素材存储根目录 |

目录结构规则：
```
{PRODUCT_BASE_DIR}/{Brand}/{ItemCode}/
├── source_image_01.jpg     # Step1 下载的原始图
├── source_image_02.jpg
├── contact_sheet.jpg       # Step6 拼合缩略图
├── main_image.jpg          # Step6 选定的主图
└── new aplus image/        # Step9 生成的 A+ 图
    ├── aplus_01.png
    └── ...
```

### 3.5 默认品牌

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `DEFAULT_BRAND` | str | "Vindhvisk" | 创建商品时的默认品牌 |

## 4. AI API 配置

### 4.1 LLM（商品描述 + A+规划 + A+脚本）

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `LLM_API_BASE` | str | https://sub2api.127space.com/v1 | API 代理地址 |
| `LLM_API_KEY` | str | "" | API Key（**必须**） |
| `LLM_MODEL` | str | "gpt-5.5" | 模型名称 |

**使用步骤**：Step 5（Listing）、Step 7（A+规划）、Step 8（A+脚本）

**协议**：OpenAI-compatible（`/v1/chat/completions`）

**SSL 注意**：sub2api 代理存在 SSL 证书问题，系统通过 `httpx.AsyncClient(verify=False)` 绕过。客户端获取方式：

```python
client = settings.get_llm_client()  # 返回 AsyncOpenAI，已处理 SSL
```

### 4.2 VLM（主图视觉分析）

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `VLM_API_BASE` | str | https://dashscope.aliyuncs.com/compatible-mode/v1 | DashScope API |
| `VLM_API_KEY` | str | "" | DashScope API Key（**必须**） |
| `VLM_MODEL` | str | "qwen3.6-plus" | 视觉模型 |

**使用步骤**：Step 6（主图分析）

**协议**：OpenAI-compatible（支持图片输入）

**获取方式**：
```python
client = settings.get_vlm_client()  # 返回 AsyncOpenAI
```

### 4.3 GPT Image（A+图片生成）

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `GPT_IMAGE_API_BASE` | str | https://ai.t8star.cn/v1 | 图片生成 API |
| `GPT_IMAGE_API_KEY` | str | "" | API Key（**必须**） |
| `GPT_IMAGE_MODEL` | str | "gpt-image-2" | 图片生成模型 |

**使用步骤**：Step 9（A+出图）

**协议**：OpenAI-compatible（`/v1/images/generations`）

## 5. 外部服务配置

### 5.1 卖家精灵

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `SELLERSPRITE_TOKEN` | str | "" | Cookie 中的 `Sprite-X-Token`（JWT） |

**使用步骤**：Step 3（关键词反查）

**获取方式**：
1. 登录 [sellerSprite.com](https://www.sellersprite.com)
2. 打开浏览器 DevTools → Application → Cookies
3. 复制 `Sprite-X-Token` 的值

**有效期**：约 24 小时，过期后需重新获取

**未配置影响**：Step 3 自动跳过，不阻塞 Pipeline

### 5.2 Chrome 浏览器

无需配置项，但需满足前提条件：

1. Chrome 已安装并运行
2. 开启 JS 权限：Chrome 菜单 → 视图 → 开发者 → ✅ 允许 Apple 事件中的 JavaScript
3. 已登录大健云仓（GigaB2B）网站

**使用步骤**：Step 1（商品采集）

| 相关配置 | 类型 | 默认值 | 说明 |
|---------|------|--------|------|
| `CHROME_LOCK_TIMEOUT` | int | 300 | Chrome 操作串行锁超时（秒） |

## 6. Pipeline 行为配置

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `STEP3_4_PARALLEL` | bool | True | Step 3/4 是否并行执行 |
| `APLUS_CONCURRENCY` | int | 5 | Step 9 A+ 图片生成并发数 |
| `POLL_INTERVAL` | int | 3 | 前端轮询间隔（秒） |

### STEP3_4_PARALLEL

- `True`（默认）：Step 3（关键词）和 Step 4（类目）通过 `asyncio.gather()` 并行执行
- `False`：串行执行 Step 3 → Step 4

### APLUS_CONCURRENCY

Step 9 生成 A+ 图片时，同时发起的 API 请求数量。建议 3~5，过高可能触发 API 限流。

### POLL_INTERVAL

前端页面轮询商品状态的间隔，用于实时展示 Pipeline 进度。

## 7. 配置检查

### 7.1 通过 API 检查

```bash
# 查看配置状态（Key 是否已填写）
curl http://localhost:8190/api/config

# 系统健康检查
curl http://localhost:8190/api/config/status
```

### 7.2 配置就绪检查清单

| 检查项 | 字段 | 影响 |
|--------|------|------|
| ✅ LLM API Key | `llm_api_configured` | Step 5/7/8 不可用 |
| ✅ VLM API Key | `vlm_api_configured` | Step 6 不可用 |
| ✅ GPT Image Key | `gpt_image_api_configured` | Step 9 不可用 |
| ⬜ SellerSprite Token | `sellersprite_configured` | Step 3 跳过（非必须） |
| ✅ 商品目录存在 | `product_dir_exists` | 图片存储 |

> 3个 AI Key 全部配置后才可运行完整 Pipeline。SellerSprite Token 为可选，缺省时 Step 3 跳过。

## 8. 常见问题

### Q: SSL 证书错误

sub2api 代理的 SSL 证书有问题，系统已内置 `verify=False` 绕过。如仍有问题，检查 `LLM_API_BASE` 是否正确。

### Q: SellerSprite Token 过期

Token 约 24h 有效。过期后 Step 3 自动跳过，不影响其他步骤。重新登录卖家精灵获取新 Token 更新 `.env` 即可。

### Q: 修改配置后生效

- `.env` 修改后需**重启后端**才能生效
- 通过 `scripts/start.sh` 启动的话，`Ctrl+C` 停止后重新执行即可

### Q: 切换 AI 模型

修改 `.env` 中对应的 `*_MODEL` 字段，例如：

```bash
LLM_MODEL=gpt-4o          # 切换 LLM 模型
VLM_MODEL=qwen2.5-vl-72b  # 切换 VLM 模型
```
