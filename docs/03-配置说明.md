# FBM铺货系统 — 配置说明

> 版本：0.1.0 | 更新日期：2026-05-13

---

## 1. 配置体系

系统使用 **pydantic-settings** 管理配置，支持3层优先级（高→低）：

1. **环境变量**（最高优先级）
2. **`.env` 文件**（`backend/.env`）
3. **代码默认值**（`config.py` 中的 `Settings` 类）

配置加载后通过 `settings = Settings()` 全局单例使用。

---

## 2. 配置文件

### 2.1 `.env` 文件

位置：`~/code/fbm-pipeline/backend/.env`

```bash
# ─── LLM API (sub2api — Listing/A+规划/A+脚本) ───
LLM_API_BASE=https://sub2api.127space.com/v1
LLM_API_KEY=sk-your-llm-key-here
LLM_MODEL=gpt-5.5

# ─── VLM API (主图分析；默认复用 LLM API 的 GPT-5.5 多模态能力) ───
VLM_API_BASE=https://sub2api.127space.com/v1
VLM_API_KEY=sk-your-vlm-key-here
VLM_MODEL=gpt-5.5
VLM_USE_LLM_API=true

# ─── GPT Image API (t8star — A+出图) ───
GPT_IMAGE_API_BASE=https://ai.t8star.cn/v1
GPT_IMAGE_API_KEY=sk-your-image-key-here
GPT_IMAGE_MODEL=gpt-image-2
GPT_IMAGE_USE_LLM_API=false

# ─── 卖家精灵 Token（Cookie中的 Sprite-X-Token）───
SELLERSPRITE_TOKEN=
```

> ⚠️ `.env` 包含敏感 Key，已加入 `.gitignore`，不入版本库。

### 2.2 环境变量覆盖

任何配置项都可通过环境变量覆盖，例如：

```bash
export LLM_API_KEY=sk-new-key
export BACKEND_PORT=9000
uvicorn app.main:app --port $BACKEND_PORT
```

## 4. AI API 配置

### 4.1 LLM（商品描述 + A+规划 + A+脚本）

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `LLM_API_BASE` | str | https://sub2api.127space.com/v1 | API 代理地址 |
| `LLM_API_KEY` | str | "" | API Key（**必须**） |
| `LLM_MODEL` | str | "gpt-5.5" | 模型名称 |

**使用步骤**：Step 5（Listing）、Step 7（A+规划）、Step 8（A+脚本）

**协议**：OpenAI-compatible（`/v1/chat/completions`）

**SSL 注意**：sub2api 代理存在 SSL 证书问题，系统通过 `httpx.AsyncClient(verify=False)` 绕过。客户端获取方式：

```python
client = settings.get_llm_client()  # 返回 AsyncOpenAI，已处理 SSL
```

### 4.2 VLM（主图视觉分析）

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `VLM_API_BASE` | str | https://sub2api.127space.com/v1 | 备用 VLM API |
| `VLM_API_KEY` | str | "" | 备用 VLM API Key |
| `VLM_MODEL` | str | "gpt-5.5" | 视觉模型 |
| `VLM_USE_LLM_API` | bool | true | 主图分析是否复用 LLM_API_BASE/LLM_API_KEY |

**使用步骤**：Step 6（主图分析）

**协议**：OpenAI-compatible（支持图片输入）

**获取方式**：
```python
client = settings.get_vlm_client()  # 返回 AsyncOpenAI
```

### 4.3 GPT Image（A+图片生成）

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `GPT_IMAGE_API_BASE` | str | https://ai.t8star.cn/v1 | 图片生成 API |
| `GPT_IMAGE_API_KEY` | str | "" | API Key（**必须**） |
| `GPT_IMAGE_MODEL` | str | "gpt-image-2" | 图片生成模型 |
| `GPT_IMAGE_USE_LLM_API` | bool | False | True 时复用 `LLM_API_BASE/LLM_API_KEY` 跑 Step 9 生图，模型仍使用 `GPT_IMAGE_MODEL` |
| `APLUS_IMAGE_API_MODE` | str | edits | Step 9 首选生图通道：`edits`/`generations` |

**使用步骤**：Step 9（A+出图）

**协议**：OpenAI-compatible（优先 `/v1/images/edits`，失败时可回退 `/v1/images/generations`）

## 5. 外部服务配置

### 5.1 卖家精灵

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `SELLERSPRITE_TOKEN` | str | "" | Cookie 中的 `Sprite-X-Token`（JWT） |

**使用步骤**：Step 3（关键词反查）

**获取方式**：
1. 登录 [sellerSprite.com](https://www.sellersprite.com)
2. 打开浏览器 DevTools → Application → Cookies
3. 复制 `Sprite-X-Token` 的值

**有效期**：约 24 小时，过期后需重新获取

**未配置影响**：Step 3 自动跳过，不阻塞 Pipeline

### 5.2 Chrome 浏览器

无需配置项，但需满足前提条件：

1. Chrome 已安装并运行
2. 开启 JS 权限：Chrome 菜单 → 视图 → 开发者 → ✅ 允许 Apple 事件中的 JavaScript
3. 已登录大健云仓（GigaB2B）网站

**使用步骤**：Step 1（商品采集）

| 相关配置 | 类型 | 默认值 | 说明 |
|---------|------|--------|------|
| `CHROME_LOCK_TIMEOUT` | int | 300 | Chrome 操作串行锁超时（秒） |

## 6. Pipeline 行为配置

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `PIPELINE_MAX_CONCURRENCY` | int | 2 | 同时真正执行的 Pipeline 任务数上限；批量启动后多余任务会在后台等待 |
| `BROWSER_WORKFLOW_CONCURRENCY` | int | 1 | 完整浏览器业务流程并发数；当前共用一个 Chrome worker tab，建议保持 1 |
| `BULK_START_MAX_TASKS` | int | 100 | 单次批量启动任务数上限 |
| `STEP1_EXTRACT_RETRY_ATTEMPTS` | int | 5 | Step 1 商品信息采集重试次数 |
| `STEP1_EXTRACT_RETRY_DELAY_SECONDS` | int | 3 | Step 1 每次重试等待秒数 |
| `STEP1_DOWNLOAD_TIMEOUT_SECONDS` | int | 300 | Step 1 素材 ZIP 下载等待超时 |
| `STEP1_MATERIAL_PACKAGE_PRIORITY` | str | To B素材包,Retail Ready素材包,Information | Step 1 素材包类型优先级，逗号分隔 |
| `STEP1_PRICE_MISSING_POLICY` | str | manual_review | 价格/成本缺失策略：fail/manual_review/continue |
| `STEP1_MATERIAL_MISSING_POLICY` | str | manual_review | 素材缺失策略：fail/manual_review/continue |
| `STEP1_ALLOW_EXISTING_MATERIALS` | bool | True | 素材下载失败时，是否允许使用商品目录中已有素材 |
| `PRICING_NET_REVENUE_RATE` | float | 0.685 | Step 2 售价扣除平台/优惠/变动费用后的净收入比例 |
| `PRICING_TARGET_MARGIN_RATE` | float | 0.05 | Step 2 目标净利率，按利润/售价计算 |
| `PRICING_MIN_PROFIT` | float | 10 | Step 2 单件最低利润 |
| `PRICING_FIXED_COST` | float | 9 | Step 2 固定成本预留 |
| `PRICING_RETURN_CREDIT_RATE` | float | 0.06 | Step 2 按货值估算的退货保险抵扣比例 |
| `STEP3_MANUAL_LOGIN_ON_AUTH_FAILURE` | bool | True | Step 3 卖家精灵未登录/过期时，打开页面等待人工登录 |
| `STEP4_MISSING_ASIN_POLICY` | str | manual_review | Step 4 缺竞品 ASIN 时的策略：fail/manual_review/continue |
| `STEP4_CATEGORY_MISSING_POLICY` | str | manual_review | Step 4 类目抓取失败时的策略：fail/manual_review/continue |
| `STEP4_ALLOW_EXISTING_CATEGORY` | bool | True | Step 4 失败时是否允许使用已有/人工类目继续 |
| `STEP5_LLM_TEMPERATURE` | float | 0.7 | Step 5 Listing 生成温度 |
| `STEP5_LLM_MAX_TOKENS` | int | 2000 | Step 5 Listing 最大输出 tokens |
| `STEP5_TITLE_MAX_CHARS` | int | 200 | Step 5 标题字符上限 |
| `STEP5_BULLET_MAX_CHARS` | int | 500 | Step 5 单条五点字符上限 |
| `STEP5_SEARCH_TERMS_MAX_BYTES` | int | 250 | Step 5 Search Terms UTF-8 字节上限；最终写入 Amazon 表时最多 20 个关键词短语，用 `, ` 分隔 |
| `STEP3_4_PARALLEL` | bool | True | Step 3/4 是否并行执行 |
| `APLUS_CONCURRENCY` | int | 1 | Step 9 A+ 图片生成并发数 |
| `POLL_INTERVAL` | int | 3 | 前端轮询间隔（秒） |

### PIPELINE_MAX_CONCURRENCY

同时运行的 Pipeline 数量。批量启动 20 个任务时，系统会先启动最多这个数量，其余任务留在后台等待名额。建议 2~3，避免本地浏览器、AI API 和磁盘任务同时拥挤。

### BROWSER_WORKFLOW_CONCURRENCY

完整浏览器流程的并发数。当前采集、类目、SellerSprite 登录态、领星查询都共用一个 Chrome worker tab，所以默认 1 最稳。只有未来改成多独立浏览器上下文时，才建议调高。

### BULK_START_MAX_TASKS

一次批量启动允许提交的任务数量上限，用于避免误选过多任务后瞬间塞满后台队列。

### STEP1 采集与素材策略

Step 1 的核心商品字段仍然是硬依赖：页面打不开、未登录或核心字段采不到时会标记失败。价格/成本缺失、素材包缺失这类可人工补救问题，默认会把任务停在 `pending_review` 并写入原因，方便补资料后继续。

`STEP1_PRICE_MISSING_POLICY` 与 `STEP1_MATERIAL_MISSING_POLICY` 支持：

- `fail`：直接标记失败
- `manual_review`：停在待人工处理
- `continue`：记录日志后继续执行

素材包下载会按 `STEP1_MATERIAL_PACKAGE_PRIORITY` 选择，例如默认优先 `To B素材包`，其次 `Retail Ready素材包`，最后 `Information`。

### STEP2 定价与利润率

Step 2 使用两个价格下限，取较高值作为建议售价：

- 目标净利率价格：确保 `利润 / 建议售价 >= PRICING_TARGET_MARGIN_RATE`
- 最低利润价格：确保单件利润不少于 `PRICING_MIN_PROFIT`

`profit_rate` 存储的是百分数数值，例如 `5.0` 表示 5%，不是 `0.05`，也不是成本回报率。

### STEP3 卖家精灵登录

默认开启 `STEP3_MANUAL_LOGIN_ON_AUTH_FAILURE`。当 Step 3 检测不到卖家精灵登录态，或 API 返回登录过期时，系统会打开卖家精灵关键词反查页面，让任务停在 `pending_review`。用户完成登录后点击“继续”，任务会从 Step 3 重试。

### STEP4 类目策略

Step 4 默认在缺竞品 ASIN 或亚马逊类目抓取失败时停在 `pending_review`，用户可以在商品详情页手动填写 Amazon 类目和叶子类目，再点击“继续”。如果商品已经有人工类目，且 `STEP4_ALLOW_EXISTING_CATEGORY=True`，系统会跳过自动抓取并继续后续步骤。

### STEP5 Listing 输出限制

Step 5 生成后会按配置自动裁剪标题、五点和 Search Terms，并把系统调整记录写入 `listing_check.system_adjustments`。商品详情页也支持人工编辑 Listing，保存后后续步骤会使用人工修正后的内容。

### STEP3_4_PARALLEL

- `True`（默认）：Step 3（关键词）和 Step 4（类目）通过 `asyncio.gather()` 并行执行
- `False`：串行执行 Step 3 → Step 4

### APLUS_CONCURRENCY

Step 9 生成 A+ 图片时，单个商品内同时发起的 API 请求数量。建议从 1 开始，过高可能触发 API 限流。

### POLL_INTERVAL

前端页面轮询商品状态的间隔，用于实时展示 Pipeline 进度。

## 7. GIGA Open API 与库存/价格同步

GIGA 商品池、库存同步和价格同步使用 Open API，不依赖 Chrome 登录态。店铺 API 地址、AK/SK、站点和履约方式只在「数据源维护」里配置，不再写入全局 `backend/.env`：

```text
GIGA_SYNC_PAGE_SIZE=200
```

动态同步要求：

- 必须显式指定 `site`，只允许 `US` 或 `JP`。
- 必须显式指定 `data_source_id`，凭据从商品数据源表读取。
- 库存事实写入 `giga_inventory`，复合唯一键为 `batch_id + site + sku_code`。
- 价格事实写入 `giga_prices`，复合唯一键为 `batch_id + site + sku_code`。
- 当前同步日志复用 `giga_sync_batches`，库存 batch 使用 `current_category=inventory_snapshot` 标识，价格 batch 使用 `current_category=price_snapshot` 标识。
- 有货/无货切换告警写入 `giga_inventory_alerts`。
- 有效成交价变化告警写入 `giga_price_alerts`，有效价按 `exclusivePrice -> discountedPrice -> price` 取值。
- `/inventory-sync` 页面展示最新库存快照，按 SKU 分页。
- Amazon 库存更新模板和普通导入表格数量覆盖读取最新 `giga_inventory.stock_qty`。
- 商品池 SKU 展开读取最新价格快照，不需要每天重拉商品详情。

手动同步：

```bash
cd /Users/liuchang/Documents/gitproject/fbm-pipeline
backend/.venv/bin/python scripts/giga_inventory_sync.py --site US --data-source-id 1 --task-id manual-giga-inventory
backend/.venv/bin/python scripts/giga_price_sync.py --site US --data-source-id 1 --task-id manual-giga-price
```

完整说明见 `docs/giga-inventory-sync.md`。

## 8. 配置检查

### 8.1 通过 API 检查

```bash
# 查看配置状态（Key 是否已填写）
curl http://localhost:8190/api/config

# 保存可编辑配置到 backend/.env（重启后生效）
curl -X PATCH http://localhost:8190/api/config \
  -H 'Content-Type: application/json' \
  -d '{"pipeline_max_concurrency":3,"browser_workflow_concurrency":1}'

# 系统健康检查
curl http://localhost:8190/api/config/status
```

前端“系统配置”页面也使用同一个保存接口。保存后会提示重启后端；这是有意设计，避免 Pipeline 并发锁、浏览器流程锁等启动期配置在运行中半途改变。

### 8.2 配置就绪检查清单

| 检查项 | 字段 | 影响 |
|--------|------|------|
| ✅ LLM API Key | `llm_api_configured` | Step 5/7/8 不可用 |
| ✅ 图片分析 API Key | `vlm_api_configured` | Step 6 不可用 |
| ✅ GPT Image Key | `gpt_image_api_configured` | Step 9 不可用 |
| ⬜ SellerSprite Token | `sellersprite_configured` | Step 3 跳过（非必须） |
| ✅ 商品目录存在 | `product_dir_exists` | 图片存储 |

> 3个 AI Key 全部配置后才可运行完整 Pipeline。SellerSprite Token 为可选，缺省时 Step 3 跳过。

## 9. 常见问题

### Q: SSL 证书错误

sub2api 代理的 SSL 证书有问题，系统已内置 `verify=False` 绕过。如仍有问题，检查 `LLM_API_BASE` 是否正确。

### Q: SellerSprite Token 过期

Token 约 24h 有效。过期后 Step 3 自动跳过，不影响其他步骤。重新登录卖家精灵获取新 Token 更新 `.env` 即可。

### Q: 修改配置后生效

- `.env` 修改后需**重启后端**才能生效
- 通过 `scripts/start.sh` 启动的话，`Ctrl+C` 停止后重新执行即可

### Q: 切换 AI 模型

修改 `.env` 中对应的 `*_MODEL` 字段，例如：

```bash
LLM_MODEL=gpt-4o          # 切换 LLM 模型
VLM_MODEL=gpt-5.5         # 切换 VLM 模型
```
